Fat-tree Construction

The fat tree network is a universal network for provably efficient communication. It was invented by Charles E. Leiserson of the Massachusetts Institute of Technology in 1985.

Fat tree network topology looks like a tree topology. In tree topology, we have same terminologies like Root, parent, child etc. This is mainly used to connect a large number of physical servers/ computers in a large data center. In a tree data structure, every branch has the same thickness. They are all skinny (low-bandwidth).

In a fat tree, branches nearer the top of the hierarchy are thicker than branches further down the hierarchy. In a telecommunications network, the branches are data links. Switches with k number of ports in a Fat tree topology is called k-ary or k-port Fat tree network topology. From the value of k, we can derive the number of core switches, aggregation switches, edge switches. A set of (k/2) number of aggregation switches and (k/2) number of edge switches are combined together and that is known as a Pod. (k/2)^2 number of core switches.

FAT-TREE Calculation



    Steps

    Let k=4, i.e. each switch has 4 ports.

  1. Number of core switches is (k/2)^2 = 4 core switches. Number of pods = k = 4.
  2. Each pod consists of (k/2)=(4/2)=2 aggregation switches and (k/2)=(4/2)=2 edge switches.
  3. Each aggregation switch within a pod is connected to (k/2)=(4/2)=2 core switches and (k/2)=(4/2)=2 edge switches.
  4. Each edge switch within a pod is connected to (k/2)=(4/2)=2 servers and (k/2)=(4/2)=2 aggregation switches. That means each pod will be connected to (k/2)^2=(4/2)^2=4 servers.

    5. Hence the maximum number of servers that can be connected to the network is (k^3)/4=16 servers.
